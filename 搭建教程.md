# ESP32-S3 音频播放器 — 从零搭建教程

> 硬件清单：ESP32-S3 N16R8 + 2.4" ST7789 TFT 240×320 屏幕 + MAX98357A 功放 + Mini SD卡模块 + 28mm 4Ω3W 喇叭 + 5个轻触按键 + 面包板 + 杜邦线
> 支持蓝牙传歌：通过手机 APP 经蓝牙 SPP 将 MP3 文件无线传输到 SD 卡

---

## 第一步：安装 Arduino IDE

1. 打开 https://www.arduino.cc/en/software 下载 **Arduino IDE 2.x**（Windows/Mac/Linux 均可）
2. 安装并打开

---

## 第二步：添加 ESP32 开发板支持

1. 打开 Arduino IDE → 菜单 **文件 → 首选项**
2. 在「附加开发板管理器网址」中填入：
   ```
   https://espressif.github.io/arduino-esp32/package_esp32_index.json
   ```
3. 点击「确定」保存
4. 菜单 **工具 → 开发板 → 开发板管理器**
5. 搜索 `esp32`，找到 **esp32 by Espressif Systems**，点击「安装」（安装最新版即可）
6. 安装完成后，菜单 **工具 → 开发板 → esp32** → 选择 **ESP32S3 Dev Module**

菜单 **工具 → 管理库**，分别搜索并安装以下两个库：

## 第四步：配置 TFT_eSPI（关键步骤！）

这一步最容易出错，请仔细操作。

### 找到配置文件

**Windows:**
```
C:\Users\你的用户名\Documents\Arduino\libraries\TFT_eSPI\User_Setup.h
```

**Mac:**
```
~/Documents/Arduino/libraries/TFT_eSPI/User_Setup.h
```

### 替换文件内容

用文本编辑器打开 `User_Setup.h`，**清空全部内容**，粘贴以下代码：

```cpp
#define USER_SETUP_INFO "ESP32-S3 ST7789 240x320"

// 驱动芯片
#define ST7789_DRIVER
#define TFT_WIDTH  240
#define TFT_HEIGHT 320

// ESP32-S3 引脚映射
#define TFT_MOSI 11
#define TFT_SCLK 12
#define TFT_CS   10
#define TFT_DC    9
#define TFT_RST   8

// 如果屏幕颜色不对（红色变蓝色），取消下面这行的注释
// #define TFT_RGB_ORDER TFT_BGR

// 加载字体
#define LOAD_GLCD
#define LOAD_FONT2
#define LOAD_FONT4
#define LOAD_FONT6
#define LOAD_FONT7
#define LOAD_FONT8
#define LOAD_GFXFF
#define SMOOTH_FONT

// SPI 速度（40MHz，稳定可靠）
#define SPI_FREQUENCY  40000000
```

保存文件。如果 IDE 正在运行，重启 Arduino IDE。

## 第五步：接线

### 总览图

```
                    ┌──────────────────────┐
                    │    ESP32-S3 开发板    │
                    │                      │
   ST7789 屏幕 ──── │  GPIO8  = RES        │
   MAX98357A  ──── │  GPIO4  = BCLK       │
   SD卡模块   ──── │  GPIO13 = MISO(共用)  │
   5个按键    ──── │  GPIO14 = CS(SD)     │
                    │  GPIO15-21 = 按键    │
                    └──────────────────────┘
```

### 详细接线表

#### 1. 屏幕（2.4" ST7789 TFT，14个引脚）

| 屏幕引脚 | ESP32-S3 引脚 | 杜邦线颜色（建议） |
|---|---|---|
| VCC | 3.3V | 红色 |
| GND | GND | 黑色 |
| SCL (SCK) | GPIO12 | 黄色 |
| SDA (MOSI) | GPIO11 | 蓝色 |
| RES (RST) | GPIO8 | 白色 |
| DC (RS) | GPIO9 | 绿色 |
| CS | GPIO10 | 橙色 |
| SDO (MISO) | GPIO13 | 紫色 |
| LED (背光) | 3.3V | 红色 |
| VCC (另一个) | 3.3V | 红色 |
| GND (另一个) | GND | 黑色 |
| 其余引脚 | 不接 | - |

> 注意：
> - LED 接 3.3V 表示背光常亮。如果不需要背光可以不接。
> - 有些 2.4寸屏幕有两组 VCC/GND，都接上更稳定。
> - SDO (MISO) 与 SD 卡模块共用 GPIO13。

#### 2. 音频功放（MAX98357A，7个引脚）

| 功放引脚 | ESP32-S3 引脚 | 说明 |
|---|---|---|
| VIN | 3.3V 或 5V | 供电（5V 声音更大） |
| GND | GND | 地 |
| BCLK | GPIO4 | 位时钟 |
| LRCLK (LRC) | GPIO5 | 左右声道时钟 |
| DIN | GPIO6 | 数据输入 |
| GAIN | 不接 | 使用默认增益 |
| SD | 不接 | 闲置 |

> 喇叭接线：喇叭的两根线接到功放模块上的**绿色端子**（SPK+ 和 SPK-），不分正负。

#### 3. SD卡模块（6个引脚）

| SD模块引脚 | ESP32-S3 引脚 | 说明 |
|---|---|---|
| VCC (3.3V) | 3.3V | 供电 |
| GND | GND | 地 |
| MISO | GPIO13 | 数据输出（与屏幕 SDO 共用） |
| MOSI | GPIO11 | 数据输入（与屏幕 SDA 共用） |
| SCK | GPIO12 | 时钟（与屏幕 SCL 共用） |
| CS | GPIO14 | 片选（独立） |

#### 4. 按键（5个，每个4脚）

每个按键的接法相同：

```
按键                ESP32-S3
----                --------
引脚A  ──────────>  对应 GPIO
引脚B  ──────────>  GND
```

| 按键 | ESP32-S3 GPIO | 功能 |
|---|---|---|
| 上 (UP) | GPIO15 | 列表上移 / 音量+ |
| 下 (DOWN) | GPIO16 | 列表下移 / 音量- |
| 确认 (OK) | GPIO17 | 播放 / 暂停 |
| 返回 (BACK) | GPIO18 | 回顶 / 上一首 |
| 菜单 (MENU) | GPIO21 | 跳转 / 返回列表 |

> 按键不分正负，对角两脚各接一根线即可。所有按键的 GND 端可以共用一根线接到面包板的 GND 排。

### 电源注意事项

- ESP32-S3 的 3.3V 引脚**同时给屏幕、SD卡、功放供电**
- 如果功放声音太大导致重启，改用 **5V** 给功放 VIN 供电
- 确保所有模块的 **GND 连在一起**（共地）

## 第六步：准备 SD 卡

1. 取一张 Micro SD 卡（8GB/16GB 均可）
2. 用电脑格式化为 **FAT32** 格式：
   - Windows：右键 SD 卡盘 → 格式化 → 文件系统选 FAT32
   - Mac：磁盘工具 → 抹掉 → MS-DOS (FAT)
3. 将 MP3/WAV/FLAC 文件直接复制到 SD 卡**根目录**（不要放子文件夹）
4. 插入 SD 卡模块

> 建议：先用 3-5 首 MP3 测试，确认能播放后再多放。

## 第七步：烧录代码

1. 用 **USB Type-C 线**连接 ESP32-S3 到电脑
2. 在 Arduino IDE 中打开 `ESP32_S3_Audio_Player.ino`
3. 菜单 **工具** 确认以下设置：

| 设置项 | 值 |
|---|---|
| 开发板 | ESP32S3 Dev Module |
| USB Mode | USB-OTG (TinyUSB) |
| Upload Mode | UART0 / Hardware CDC |
| Flash Size | 16MB (128Mb) |
| PSRAM | OPI PSRAM 8MB |
| Partition Scheme | **Huge APP (3MB No OTA/1MB SPIFFS)** |
| Port | 选择对应的 COM 口或串口设备 |

4. 点击左上角 **→ 上传按钮**（箭头图标）
5. 等待编译和上传完成（底部状态栏显示「上传完成」）

> 如果上传失败，按住 ESP32-S3 板上的 **BOOT 按钮**，然后按一下 **RST 按钮**，松开 BOOT，重新上传。

## 常见问题排查

### 问题1：屏幕不显示 / 全黑

| 检查项 | 操作 |
|---|---|
| 接线 | 确认 VCC 接 3.3V，GND 接地 |
| 引脚 | 确认 SCL→GPIO12, SDA→GPIO11, DC→GPIO9, RES→GPIO8, CS→GPIO10 |
| User_Setup.h | 确认已正确配置（见第四步），特别是 TFT_HEIGHT=320 |
| LED 背光 | 确认 LED 接了 3.3V（2.4寸屏幕背光引脚是 LED 不是 BLK） |

### 问题2：屏幕颜色不对（红蓝反转）

打开 `User_Setup.h`，取消这行注释：
```cpp
#define TFT_RGB_ORDER TFT_BGR
```
重新上传代码。

### 问题3：没有声音

| 检查项 | 操作 |
|---|---|
| 功放接线 | 确认 BCLK→GPIO4, LRCLK→GPIO5, DIN→GPIO6 |
| 喇叭接线 | 确认喇叭线接在 MAX98357A 的绿色端子上 |
| 音量 | 确认音量不为 0（播放界面按上键加音量） |
| 供电 | 功放 VIN 接 5V 试试（3.3V 可能供电不足） |

### 问题4：SD卡读取失败

| 检查项 | 操作 |
|---|---|
| 格式 | 确认 SD 卡是 **FAT32** 格式（不是 exFAT 或 NTFS） |
| 接线 | 确认 CS→GPIO14, MISO→GPIO13（与屏幕 SDO 共用） |
| 卡片 | 换一张卡试试，有些劣质卡兼容性差 |
| 模块 | 确认买的是 3.3V 模块，不是 5V 版本 |

### 问题5：编译报错

| 错误信息 | 解决方案 |
|---|---|
| `TFT_eSPI.h: No such file` | 确认已安装 TFT_eSPI 库 |
| `Audio.h: No such file` | 确认已安装 ESP32-audioI2S 库 |
| `ST7789_DRIVER not defined` | 确认 User_Setup.h 配置正确 |
| 内存不足 | 菜单 工具 → Partition Scheme → 选 `Huge APP (3MB No OTA/1MB SPIFFS)` |
| PSRAM 相关错误 | 确认 工具 → PSRAM → 选 `OPI PSRAM 8MB` |

### 问题6：上传失败 / 无法连接

| 检查项 | 操作 |
|---|---|
| USB 线 | 确认是**数据线**不是充电线 |
| 端口 | 确认 工具 → Port 选了正确的串口 |
| BOOT 模式 | 按住 BOOT → 按一下 RST → 松开 BOOT → 重新上传 |
| 驱动 | 如果电脑没识别到串口，安装 CH340/CP2102 驱动 |

## 第九步：蓝牙传歌功能

ESP32 固件已内置蓝牙 SPP（经典蓝牙串口透传），手机通过 APP 可将 MP3 文件无线传输到 SD 卡。

### 传输流程

```
手机选择MP3文件 → 蓝牙SPP连接 → 二进制协议传输 → ESP32写入SD卡 → 自动刷新列表
```

### 蓝牙设备名

ESP32 上电后蓝牙自动启动，设备名为 **ESP32_MusicBox**，手机蓝牙设置中搜索该名称即可配对。

### 传输协议

```
手机 → ESP32:  [0xAB] [文件名 UTF-8] [0x00] [文件大小 4字节小端] [文件内容 N字节]
ESP32 → 手机:  收到完整文件后回复 "OK\n"，出错回复 "ERR\n"
```

### 蓝牙遥控命令

在未传输文件时，手机可发送单字节命令控制播放：

| 命令字节 | 功能 | ESP32 回复 |
|---|---|---|
| 0xC0 | 播放/暂停 | OK\n |
| 0xC1 | 上一首 | OK\n |
| 0xC2 | 下一首 | OK\n |
| 0xC3 | 音量+ | OK\n |
| 0xC4 | 音量- | OK\n |
| 0xC5 | 获取状态 | JSON 状态字符串\n |

### 蓝牙时间同步

| 命令 | 格式 |
|---|---|
| 0xD0 + 时间戳 | [0xD0] [4字节 Unix时间戳大端序] |

同步后列表界面右上角显示时钟和年月日。

### ESP32 端传输界面

传输过程中 ESP32 屏幕自动切换到接收界面，显示：
- 文件名
- 进度条和百分比（大号字体）
- 已接收 / 总大小
- 传输完成后 2 秒自动返回列表

> 注意：传输中可按 **返回键** 取消接收。传输速度约 25-37 KB/s，一首 5MB MP3 约需 3 分钟。

### 编译蓝牙版代码的重要设置

蓝牙功能需要更多 Flash 空间，必须在 Arduino IDE 中调整：

| 设置项 | 值 |
|---|---|
| Partition Scheme | **Huge APP (3MB No OTA/1MB SPIFFS)** |
| PSRAM | **OPI PSRAM 8MB** |

否则会报编译内存不足错误。

## 第十步：使用 Android APP 传歌

### 方法一：编译 Android APP（推荐）

项目位于 `AndroidApp/` 目录，使用 **Android Studio** 打开。

1. 安装 Android Studio（https://developer.android.com/studio）
2. 打开 `ESP32_Audio_Player/AndroidApp/` 目录
3. 等待 Gradle 同步完成
4. 连接手机（开发者模式 + USB调试）
5. 点击绿色 ▶ 按钮编译并安装到手机

### 方法二：手机直接编译（无需电脑）

详见 `AndroidApp/手机编译APK教程.md`，通过 GitHub Actions 自动编译。

### 方法三：直接下载 APK

仓库根目录的 `ESP32_MusicBox.apk` 可直接下载安装到 Android 手机。

### APP 使用步骤

1. 打开 ESP32 MusicBox APP
2. 确保手机蓝牙已打开
3. 点击「扫描设备」按钮
4. 在列表中找到 **ESP32_MusicBox**，点击连接
5. 连接成功后显示遥控面板和时间同步按钮
6. 点击「选择音乐文件」选择 MP3/WAV/FLAC 文件
7. 点击「发送到设备」
8. 等待传输完成（显示进度条）
9. ESP32 端自动刷新文件列表

### APP 遥控功能

连接成功后可使用以下遥控按钮：
- 播放/暂停、上一首、下一首
- 音量加、音量减
- 同步时间（将手机当前时间同步到 ESP32）

### 蓝牙常见问题

| 问题 | 解决方案 |
|---|---|
| 手机搜不到 ESP32_MusicBox | 确认 ESP32 已上电，代码已上传蓝牙版；手机蓝牙关闭再打开重试 |
| 连接后断开 | 靠近 ESP32（< 5米），避免 Wi-Fi 2.4G 干扰 |
| 传输中断 | 文件可能太大，建议单文件 < 20MB；检查 SD 卡写入速度 |
| 传输后文件不可播放 | 可能传输不完整，检查 ESP32 串口日志是否有 `[BT] 接收完成` 输出 |

## 引脚速查卡

```
ESP32-S3 GPIO 分配总表（2.4" ST7789 版）
═══════════════════════════════════════
GPIO 4  → MAX98357A BCLK
GPIO 5  → MAX98357A LRCLK
GPIO 6  → MAX98357A DIN
GPIO 8  → ST7789 RES (RST)
GPIO 9  → ST7789 DC (RS)
GPIO 10 → ST7789 CS
GPIO 11 → ST7789 SDA (MOSI) ← 共享
GPIO 12 → ST7789 SCL (SCK)  ← 共享
GPIO 13 → ST7789 SDO (MISO) + SD卡 MISO ← 共享
GPIO 14 → SD卡 CS
GPIO 15 → 按键 UP
GPIO 16 → 按键 DOWN
GPIO 17 → 按键 OK
GPIO 18 → 按键 BACK
GPIO 21 → 按键 MENU
3.3V    → 屏幕 VCC + LED + SD卡 VCC + 功放 VIN
GND     → 所有模块共地
═══════════════════════════════════════
```
